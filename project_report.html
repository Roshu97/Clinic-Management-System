Clinic Management System - Project Report

1. Project Overview

The Clinic Management System is a web-based application designed to manage the operations of a clinic, providing separate portals for doctors and receptionists. It aims to streamline patient management, scheduling, and other administrative tasks.

2. Technology Stack

  • Frontend: HTML, CSS, JavaScript
  • Backend/Authentication: Google Firebase (Authentication, Firestore for data storage)
  • Backend (Local Development): json-server (used in backend/package.json)
  • Server: serve (used in root package.json for serving static files)
  • Cloud Functions: Firebase Functions (Node.js 22, firebase-admin, firebase-functions)

3. Project Structure

  • assets/: Shared assets like firebase-init.js for Firebase configuration, and other static files (e.g., clinic-hall.svg). Also contains Firestore data scripts.
  • login/: Contains login and registration forms for different user roles (doctor, receptionist).
    • login/doctor/: Doctor login page.
    • login/doctor_registration/: Doctor registration page.
    • login/receptionist/: Receptionist login page.
    • login/receptionist_registration/: Receptionist registration page.
  • doctor/: Doctor's portal functionalities (e.g., billing, dashboard, history, patients, prescriptions, profile, queue).
  • receptionist/: Receptionist's portal functionalities (e.g., dashboard).
  • backend/: Contains json-server setup and potentially other backend logic.
  • functions/: Firebase Cloud Functions.
  • dataconnect/: Data Connect related configurations and schema.
  • index.html, index.css, index.js: The main landing page of the application.

4. Key Features

  • User Authentication: Separate login and registration for Doctors and Receptionists using Firebase Authentication.
  • Doctor Portal: Access to patient records, prescription management, billing, dashboard, and queue management.
  • Receptionist Portal: Patient queue management, registration, and billing.
  • Data Management: Utilizes Firebase Firestore for data storage (though some parts are marked as "not fully implemented yet" in the README).
  • Static File Serving: Frontend served via a simple Node.js `serve` package.

5. Firebase Integration Details

Firebase is a core component of this system, primarily for authentication and data storage. The `assets/firebase-init.js` file initializes the Firebase application and exports the `app`, `auth`, and `db` (Firestore) objects for use throughout the application.

  • Authentication: `createUserWithEmailAndPassword` is used for user registration. `onAuthStateChanged` is used to monitor the user's authentication state.
  • Firestore: Used for storing patient data, billing information, prescriptions, and token data. Examples of `addPatient` and `getAllPatients` functions are present, indicating basic CRUD operations with Firestore.
  • Firebase Functions: The `functions/` directory suggests the use of Firebase Cloud Functions for server-side logic, though specific implementations were not detailed in the initial overview.

6. Workflow and Execution

6.1. Firebase Project Setup

To run the project, a Firebase project needs to be set up, and the `firebaseConfig` object from the Firebase Console must be updated in `assets/firebase-init.js`.

6.2. Running the Frontend Server

The frontend is served using a Node.js server. After installing dependencies (`npm install`), the server can be started with `npm start`, typically running on `http://localhost:5000` (or another available port).

6.3. Usage

Users can navigate to `http://localhost:9003/homepage.html` to access the application. From there, they can log in or register for either the Doctor or Receptionist portals.

7. Conclusion

The Clinic Management System provides a foundational web application for managing clinic operations with distinct user roles. Leveraging Firebase for authentication and data storage, it offers a scalable and robust backend solution. The modular structure allows for clear separation of concerns between different user portals and backend functionalities.
